================================================================================
 NuCRM SaaS — SUPERADMIN TOKEN & USAGE CONTROL SYSTEM
 Complete System for Managing AI Tokens, API Keys, Rate Limits, and Costs
 Per Tenant, Per User, Per Module
================================================================================

WHAT THIS SOLVES:
  Without token control, one customer can:
  - Burn through your entire OpenAI budget in a day
  - Cost you more than they pay you
  - Get your API key rate-limited (affecting ALL customers)
  - Abuse AI features without limits

THIS SYSTEM GIVES SUPERADMIN:
  1. Global token budget (total OpenAI/WhatsApp/Twilio spend cap)
  2. Per-tenant token limits (how much AI each customer can use)
  3. Per-user limits within a tenant
  4. Per-module limits (scoring, WhatsApp, voice, content separately)
  5. Real-time usage dashboard (who's spending what)
  6. Auto-suspend when limit hit
  7. Cost alerts at 50%, 80%, 100% of budget
  8. API key rotation
  9. Usage anomaly detection (sudden spike = alert)
  10. Billing reconciliation (what was charged vs what was used)

 PART B: HOW TOKEN CONTROL WORKS — STEP BY STEP
================================================================================

SCENARIO 1: Superadmin Sets Global OpenAI Budget
═══════════════════════════════════════════════════════════════════════════════

  Superadmin goes to: /superadmin/token-control

  Sets:
    OpenAI monthly budget: ₹10,000 (10,00,000 paise)
    Alert at 50%: ✓
    Alert at 80%: ✓
    Hard cap at 100%: ✓ (stop all AI when budget exhausted)

  System behavior:
    - Every OpenAI API call → cost added to current_month_cents
    - At ₹5,000 (50%) → send alert to superadmin
    - At ₹8,000 (80%) → send alert, warn tenants
    - At ₹10,000 (100%) → BLOCK all new OpenAI calls
      → Return: "AI budget exhausted. Contact support or wait for next month."
    - On 1st of next month → reset current_month_cents to 0


SCENARIO 2: Superadmin Sets Per-Tenant Limits
═══════════════════════════════════════════════════════════════════════════════

  Superadmin sees: "Tenant ABC used 95% of their AI budget this month"

  Goes to tenant detail → Sets limits:
    AI scores/month: 1,000
    WhatsApp msgs/month: 500
    Voice mins/month: 30
    Content pieces/month: 20
    Total monthly cost: ₹5,000
    Hard cap action: "block" (stop when limit hit)

  System behavior:
    - Every AI call → check tenant's remaining limit
    - If tenant hits limit → block new calls, show upgrade prompt
    - Tenant sees: "You've used 1000/1000 AI scores. Upgrade to continue."


SCENARIO 3: Per-User Limits (Within a Tenant)
═══════════════════════════════════════════════════════════════════════════════

  Tenant admin wants to prevent one salesperson from burning the budget.

  Sets for user "Rahul":
    AI scores/day: 50
    WhatsApp msgs/day: 20
    Max cost per call: ₹10

  System behavior:
    - Rahul scores 50 leads → hits daily limit
    - 51st score → "Daily limit reached. Try tomorrow."
    - Prevents one user from consuming the entire tenant budget


SCENARIO 4: Anomaly Detection
═══════════════════════════════════════════════════════════════════════════════

  System monitors every tenant's daily spending.

  Normal pattern: Tenant ABC spends ₹200/day on AI
  Today: Tenant ABC spent ₹5,000 on AI

  Anomaly detected:
    - Deviation: 2400% above normal
    - Cause analysis: "bulk_scoring" (scoring 500 contacts in 1 hour)
    - Action: Send alert to superadmin, throttle to 10 requests/min
    - Superadmin reviews: "Legitimate — they imported 500 new leads"
    → Mark as reviewed, increase limit temporarily

  If suspicious:
    - Cause: "loop" (same request repeated 1000x)
    - Action: BLOCK immediately, alert superadmin
    - Possible API key leak or bug


SCENARIO 5: API Key Rotation
═══════════════════════════════════════════════════════════════════════════════

  Superadmin needs to rotate an OpenAI key (security concern or rate limit).

  1. Generate new OpenAI key
  2. Add to api_keys_registry:
     - service: 'openai'
     - encrypted_key: [encrypted new key]
     - is_active: true
     - is_primary: false (not yet)
  3. Test new key with a small request
  4. Set new key as primary: is_primary = true
  5. Set old key as inactive: is_active = false
  6. System now uses new key for all requests
  7. Old key kept for reference (can rollback)

 PART D: API — Token Control Endpoints
================================================================================

All endpoints require: Superadmin authentication

GLOBAL BUDGET MANAGEMENT:
  GET    /api/superadmin/token-control/budgets
         → List all service budgets with current usage

  PUT    /api/superadmin/token-control/budgets/{service}
         → Set global budget for a service
         Body: { monthly_budget_cents, alert_50, alert_80, hard_cap }

  POST   /api/superadmin/token-control/budgets/reset
         → Reset current month spending (manual override)
         Body: { service }

API KEY MANAGEMENT:
  GET    /api/superadmin/token-control/keys
         → List all API keys (masked, with usage)

  POST   /api/superadmin/token-control/keys
         → Add new API key
         Body: { service, key, key_name, monthly_budget_cents }

  PUT    /api/superadmin/token-control/keys/{id}
         → Update key (set primary, toggle active, rotate)
         Body: { is_primary, is_active }

  DELETE /api/superadmin/token-control/keys/{id}
         → Delete a key

TENANT LIMIT MANAGEMENT:
  GET    /api/superadmin/token-control/tenants
         → List all tenants with usage vs limits

  GET    /api/superadmin/token-control/tenants/{tenantId}
         → Detailed usage for one tenant

  PUT    /api/superadmin/token-control/tenants/{tenantId}/limits
         → Set per-tenant limits
         Body: { openai_monthly_limit, whatsapp_monthly_msgs, ... }

  POST   /api/superadmin/token-control/tenants/{tenantId}/reset
         → Reset tenant's usage counters
         Body: { reason }

USER LIMIT MANAGEMENT:
  GET    /api/superadmin/token-control/users?tenantId=xxx
         → List per-user limits for a tenant

  PUT    /api/superadmin/token-control/users/{userId}/limits
         → Set per-user limits
         Body: { module, daily_limit, monthly_limit, max_cost_per_call }

ALERTS & ANOMALIES:
  GET    /api/superadmin/token-control/alerts
         → List all alerts (with filters)

  PUT    /api/superadmin/token-control/alerts/{id}/acknowledge
         → Acknowledge an alert

  GET    /api/superadmin/token-control/anomalies
         → List detected anomalies

  PUT    /api/superadmin/token-control/anomalies/{id}/review
         → Mark anomaly as reviewed
         Body: { action_taken, notes }

REPORTING:
  GET    /api/superadmin/token-control/report?period=2026-04
         → Full usage report for a billing period

  GET    /api/superadmin/token-control/report/export?format=csv
         → Export usage data as CSV

  GET    /api/superadmin/token-control/report/top-spenders
         → Top 10 spending tenants

 PART E: HOW IT INTEGRATES INTO EXISTING AI MODULES
================================================================================

Every AI module call goes through a CHECK before making the actual API call:

  // In lib/ai/common.ts — shared token check function
  async function checkTokenAndLimits(tenantId, userId, module, estimatedCost) {
    // 1. Check global budget
    const globalBudget = await getGlobalBudget(module.service);
    if (globalBudget.hard_cap_enabled &&
        globalBudget.current_month_cents >= globalBudget.monthly_budget_cents) {
      throw new Error('PLATFORM_BUDGET_EXHAUSTED');
    }

    // 2. Check tenant limits
    const tenantLimits = await getTenantLimits(tenantId);
    if (tenantLimits) {
      const usage = await getCurrentUsage(tenantId, module.name);
      if (usage.monthly_count >= tenantLimits.monthly_limit) {
        throw new Error('TENANT_LIMIT_EXCEEDED');
      }
      if (tenantLimits.total_monthly_cost >= 0 &&
          usage.monthly_cost_cents >= tenantLimits.total_monthly_cost) {
        throw new Error('TENANT_COST_LIMIT_EXCEEDED');
      }
    }

    // 3. Check user limits
    const userLimits = await getUserLimits(tenantId, userId, module.name);
    if (userLimits) {
      const dailyUsage = await getDailyUsage(tenantId, userId, module.name);
      if (dailyUsage.count >= userLimits.daily_limit) {
        throw new Error('USER_DAILY_LIMIT_EXCEEDED');
      }
      if (estimatedCost > userLimits.max_cost_per_call) {
        throw new Error('CALL_TOO_EXPENSIVE');
      }
    }

    // 4. Check for anomalies
    const anomaly = await checkForAnomaly(tenantId, module.service, estimatedCost);
    if (anomaly) {
      // Throttle or block based on anomaly severity
      await applyAnomalyAction(anomaly);
    }

    // All checks passed — proceed with API call
    return { allowed: true };
  }

  // After API call — record usage
  async function recordUsage(tenantId, userId, module, costCents, tokensUsed) {
    // Update global budget
    await incrementGlobalBudget(module.service, costCents);

    // Update tenant usage
    await incrementTenantUsage(tenantId, module.name, costCents, tokensUsed);

    // Update user usage
    await incrementUserUsage(tenantId, userId, module.name, costCents);

    // Check if any thresholds were crossed
    await checkThresholdAlerts(tenantId, module.service);

    // Log individual call
    await logAICall(tenantId, userId, module, costCents, tokensUsed);
  }

Every AI module wraps its API call:

  // Before:
  const response = await openai.chat.completions.create({ ... });

  // After:
  await checkTokenAndLimits(tenantId, userId, 'lead_scoring', estimatedCost);
  const response = await openai.chat.completions.create({ ... });
  await recordUsage(tenantId, userId, 'lead_scoring', actualCost, tokensUsed);
